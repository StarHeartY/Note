## 1 类与对象

### 1.1 类的基础结构

-   **访问控制**
    -   `public`：类内外均可访问
    -   `private`：仅类内和友元可访问（类默认权限）
    -   `protected`：类内、派生类、友元可访问
-   **`this` 指针**
    -   每个非静态成员函数都隐式含有一个 `this` 指针，指向调用该函数的对象
    -   在成员函数内访问任何成员，实际上都是通过 `this->member` 完成的
-   **`const` 成员函数**
    -   声明方式：`返回类型 函数名(参数) const;`
    -   含义：该函数不会修改任何成员变量（`mutable` 修饰的除外）
    -   `const` 对象只能调用 `const` 成员函数；非 `const` 对象优先调用非 `const` 版本
-   **`static` 成员**
    -   **静态数据成员**：属于类本身，所有对象共享同一份。必须在类外单独定义（在 `.cpp` 文件中 `类型 类名::变量名 = 值;`）
    -   **静态成员函数**：属于类，没有 `this` 指针，只能访问静态成员

### 1.2 构造函数与析构函数

-   **构造函数**
    -   与类同名、无返回值，在对象创建时自动调用
    -   可以有参数，支持重载
-   **初始化列表**
    -   语法：`构造函数名(参数) : 成员1(值1), 成员2(值2) { 函数体 }`
    -   执行顺序取决于成员的声明顺序，与初始化列表书写顺序无关
    -   **必须使用的场景**：`const` 成员、引用成员、无默认构造函数的成员对象、基类的带参构造
-   **委托构造函数**
    -   一个构造函数调用本类的另一个构造函数
    -   语法：`A(int x) : A(x, 0) {}`
-   **析构函数**
    -   名称：`~类名()`，无返回值，无参数，不可重载
    -   对象生命周期结束时自动调用
    -   **调用顺序**：与构造函数严格相反（先构造的后析构，后构造的先析构）

**例：**
```cpp
class Apartment {
private:
    string residentName; // 租客姓名
    int* networkCable;   // 模拟需要动态分配的资源（比如拉一条私人网线）

public:
    // 1. 构造函数 (Constructor)
    // 与类同名，没有返回值。在对象被创建时【自动】被调用。
    Apartment(string name) {
        residentName = name;
        // 模拟入住时配置资源：在堆区动态申请内存
        networkCable = new int[100]; 
        
        cout << ">>> [入住通知] 构造函数被调用！" << endl;
        cout << residentName << " 搬进了公寓，并接通了私人网线。" << endl;
    }

    // 2. 析构函数 (Destructor)
    // 名字是在类名前加波浪号 ~，没有返回值，且【绝对不能有参数】。
    // 在对象的生命周期结束时【自动】被调用。
    ~Apartment() {
        // 退房时必须清理资源：释放堆区内存，防止内存泄漏
        delete[] networkCable; 
        
        cout << "<<< [退房通知] 析构函数被调用！" << endl;
        cout << residentName << " 退租了，网线已拆除，房间已清理。" << endl;
    }

    // 一个普通的成员函数
    void writeCode() {
        cout << residentName << " 正在公寓里快乐地敲 C++ 代码..." << endl;
    }
};

int main() {
    cout << "------ 故事开始 ------" << endl;
    
    { // 注意这对大括号，它人为划定了一个局部的“作用域”
        Apartment myRoom("易睿"); // 运行到这一行时，对象被创建，自动触发构造函数
        myRoom.writeCode();
    } // 运行到这个右大括号时，myRoom 对象的生命周期结束了。
      // 此时编译器会【自动触发析构函数】。
    
    cout << "------ 故事结束 ------" << endl;
    
    return 0;
}
```

### 1.3 拷贝控制

-   **拷贝构造函数**
    -   语法：`类名(const 类名 &其他);`
    -   调用时机：新对象从已有对象初始化时（值传递、返回局部对象）
    -   **浅拷贝**：逐字节复制指针，导致新旧对象指向同一内存（重复释放的危险）
    -   **深拷贝**：为指针成员分配新内存，并复制内容
-   **拷贝赋值运算符**
    -   语法：`类名& operator=(const 类名 &其他);`
    -   需要注意**自赋值保护**：`if (this == &其他) return *this;`
    -   同样需要深拷贝，步骤：释放旧资源 → 分配新资源 → 复制数据
-   **三/五法则**
    -   如果自定义了析构函数、拷贝构造或拷贝赋值中的任意一个，通常三个都需要自定义
-  **右值引用 (`&&`)**：
    -   用于绑定到一个将要销毁的对象（如临时变量或字面量），表示该对象的资源可以被“安全偷取”。
- **移动构造与移动赋值**：
    -   C++11 引入的另外两个拷贝控制成员。它们不分配新内存，而是直接接管源对象（右值）的底层指针，然后将源对象的指针置空，从而极大提升动态资源转移的性能。
- **`std::move` 的本质**：
    -   它本身不移动任何东西，仅仅是一个强制类型转换，将左值无条件转换为右值引用，告诉编译器“这个对象我以后不用了，你可以触发移动语义”。

## 2 继承与多态

### 2.1 继承机制

-   **三种继承方式**
    -   `public` 继承：基类 `public` → 派生类 `public`，基类 `protected` → 派生类 `protected`
    -   `protected` 继承：基类 `public/protected` → 派生类 `protected`
    -   `private` 继承：基类 `public/protected` → 派生类 `private`
    -   基类的 `private` 成员在任何继承方式下派生类都不可直接访问
-   **派生类对象的构造顺序**
    -   基类构造函数 → 派生类成员对象的构造函数（按声明顺序）→ 派生类构造函数体
    -   **析构顺序完全相反**
-   **同名隐藏**
    -   派生类中定义了与基类同名的函数（即使参数不同），基类该名字的所有重载版本在派生类中都被隐藏
    -   如需访问基类被隐藏的同名函数：`派生类对象.基类名::函数名(参数)` 或 在派生类中使用 `using 基类名::函数名;`

### 2.2 多态与虚函数

-   **静态绑定 vs 动态绑定**
    -   **静态绑定**（编译期）：根据指针/引用的**静态类型**决定调用哪个函数（非虚函数）
    -   **动态绑定**（运行期）：根据指针/引用所指向的**实际对象类型**决定调用（虚函数）
    -   动态绑定必须满足：基类指针或引用调用虚函数
-   **`virtual` 与 `override`**
    -   `virtual`：声明虚函数，基类中声明后，派生类中自动为虚
    -   `override`：显式标注该函数意图重写基类虚函数，编译器会检查是否正确重写
-   **虚函数表（vtable）与虚指针（vptr）**
    -   每个含虚函数的类有一张虚函数表，存储该类所有虚函数的地址
    -   每个对象有一个隐式虚指针，指向该类的虚函数表
    -   **单继承模型**：派生类的虚表中，重写的虚函数会覆盖基类对应槽位；新增虚函数追加在末尾
-   **虚析构函数的重要性**
    -   如果基类指针指向派生类对象，通过它 `delete`，而基类析构函数不是虚函数，则只调用基类析构，派生类资源泄漏
    -   解决方案：将基类析构函数声明为 `virtual`
-   **纯虚函数与抽象类**
    -   纯虚函数：`virtual 返回类型 函数名() = 0;`
    -   含有纯虚函数的类为抽象类，**不能实例化**
    -   派生类若未重写全部纯虚函数，仍为抽象类

## 3 模板与STL

### 3.1 函数模板

-   **定义**
    -   `template <typename T> T max(T a, T b) { return a > b ? a : b; }`
    -   编译器根据调用时的实参类型自动推导 `T`，生成对应的函数实例
-   **模板特化**
    -   **全特化**：为特定类型提供专门实现
    -   `template <> const char* max(const char* a, const char* b) { ... }`
**【例】：**
```cpp
// 1. 先给出通用的模板（基础款）
template <typename T>
T myMax(T a, T b) {
    return (a > b) ? a : b;
}

// 2. 针对 const char* 提供特化版（VIP定制款）
// template <> 表示这是一个特化版本，没有任何需要推导的泛型参数了
template <> 
const char* myMax<const char*>(const char* a, const char* b) {
    // 使用 strcmp 进行字符串内容的字典序比较
    return (strcmp(a, b) > 0) ? a : b; 
}
```
### 3.2 类模板

-   **定义**
    -   `template <typename T> class Stack { T data[100]; int top; ... };`
    -   使用：`Stack<int> s;`，必须显式指定模板参数
-   **非类型模板参数**
    -   `template <typename T, int N> class Array { T data[N]; };`
    -   参数必须是编译期常量，如 `Array<int, 10> a;`
-   **类外定义成员函数**
    -   `template <typename T> void Stack<T>::push(const T &val) { ... }`

### 3.3 STL 核心容器

-   **`vector`（动态数组）**
    -   底层是连续内存的动态数组
    -   **扩容机制**：`size()` 达到 `capacity()` 时，申请更大的内存（通常1.5倍或2倍），拷贝/移动原有元素，释放旧内存
    -   所有指向原内存的迭代器、指针、引用在扩容后**全部失效**
    -   `resize(n)`：改变元素个数，多则构造/少则析构
    -   `reserve(n)`：只改变容量，不改变元素个数
-   **`list`（双向链表）**
    -   底层是非连续内存的双向链表，支持高效的任意位置插入/删除
    -   删除元素时，仅被删元素的迭代器失效，其他迭代器不受影响
-   **`map` vs `unordered_map`**
    -   `map`：底层为红黑树，元素按 `key` 排序，查找/插入/删除时间复杂度 `O(log n)`
    -   `unordered_map`：底层为哈希表，元素无序，平均时间复杂度 `O(1)`，最坏 `O(n)`

### 3.4 迭代器与算法

-   **迭代器分类（按能力递增）**
    -   **输入/输出**：只读或只写，单次遍历（常用于流）
    -   **前向**：可多次读写，只能向前移动（`forward_list`）
    -   **双向**：可前进和后退（`list`, `map`, `set`）
    -   **随机访问**：支持 `+n`, `-n`, `[]` 等操作（`vector`, `deque`, `array`, `string`）
-   **迭代器失效**
    -   `vector`：插入元素导致扩容时，所有迭代器失效；删除元素时，被删元素及之后元素的迭代器失效
    -   `list`：插入元素从不失效；删除元素仅被删元素的迭代器失效
-   **常用算法（`<algorithm>`）**
    -   `sort(begin, end)`：默认升序排序，要求随机访问迭代器
    -   `sort(begin, end, comp)`：自定义比较函数（或函数对象/Lambda）
    -   `find(begin, end, value)`：返回第一个等于 `value` 的迭代器，未找到返回 `end`

## 4 异常处理

### 4.1 基本机制

-   **语法**
    -   `try { ... throw 异常对象; ... } catch (异常类型 变量) { ... }`
    -   抛出异常后，`try` 块内后续代码不再执行，控制权转至 `catch`

### 4.2 栈展开

-   **定义**：异常抛出后，函数沿调用链反向（由内向外）查找匹配的 `catch` 子句
-   **析构调用**：在栈展开过程中，所有离开作用域的局部（自动）对象都会被自动调用析构函数，确保资源不泄漏
-   **出题标志**：在构造/析构函数中打印信息，判断异常发生后对象的析构顺序

### 4.3 捕获规则

-   **类型匹配**
    -   `catch` 按书写顺序依次尝试匹配
    -   派生类异常对象可被基类异常类型的 `catch` 捕获
    -   `catch(...)` 捕获所有类型，**必须放在所有 `catch` 块的最后**，否则编译报错
-   **重新抛出**
    -   `throw;`：在 `catch` 中使用，将捕获的异常原样重新抛出，保留原始异常类型
    -   `throw e;`：抛出当前 `catch` 收到的异常对象的拷贝，可能导致对象切片（若 `e` 是基类引用，则丢失派生类信息）

### 4.4 构造与析构的异常

-   **构造函数中的异常**
    -   构造函数抛出异常意味着对象构造失败，该对象的析构函数**不会被调用**
    -   但所有已经构造完毕的成员对象和基类子对象会正常析构
-   **析构函数绝不能抛出异常**
    -   如果析构函数在栈展开过程中又抛出异常，C++ 将直接调用 `std::terminate()` 终止程序

## 5 文件与流

### 5.1 标准I/O与流状态

-   **流状态标志**
    -   `good()`：流完全正常，`true`
    -   `eof()`：到达文件末尾，`true`
    -   `fail()`：读写格式错误等可恢复错误，`true`
    -   `bad()`：系统级不可恢复错误，`true`
    -   判断习惯：直接用 `while (流对象 >> 变量)`，因为流对象可隐式转换为 `bool`（等价于 `!fail()`）
-   **格式化**
    -   `setw(n)`：设置输出宽度（需 `#include <iomanip>`）
    -   `setprecision(n)`：设置浮点数精度

### 5.2 文件流操作

-   **文件流对象**
    -   `ifstream`：读文件
    -   `ofstream`：写文件
    -   `fstream`：读写文件
-   **打开文件**
    -   方式一：构造时传参 `ifstream in("data.txt");`
    -   方式二：先创建，再调用 `in.open("data.txt");`
    -   使用前检查：`if (!in.is_open()) { /* 错误处理 */ }`
-   **打开模式（可组合，如 `ios::in | ios::binary`）**
    -   `ios::in`：读模式
    -   `ios::out`：写模式（默认清空文件）
    -   `ios::binary`：二进制模式
    -   `ios::app`：追加模式，写入内容接在原文件末尾

### 5.3 文件读写

-   **文本文件**
    -   读取：`>>` 按空白分隔读取；`getline(in, str)` 一次读一行（不保留换行符）
    -   典型读循环：`while (in >> data)` 或 `while (getline(in, line))`
-   **二进制文件**
    -   写入：`out.write( (char*)&变量, sizeof(变量) );`
    -   读取：`in.read( (char*)&变量, sizeof(变量) );`
    -   必须以 `ios::binary` 模式打开，否则可能因特殊字符被转义而错误
