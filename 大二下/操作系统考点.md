### 👑 绝对核心：问答题全考点精析（直接背诵版）

这 9 道问答题涵盖了操作系统的核心骨架，考试遇到直接把下面的关键词写上去。

1. 为什么要引入线程？

- **传统进程的痛点**：进程是资源分配和调度的基本单位，创建、销毁和切换进程时的时空开销非常大。
    
- **引入线程的目的**：为了减少程序并发执行时所付出的时空开销，提高操作系统的并发性能。线程作为独立调度的基本单位，而进程只作为资源分配的单位，线程切换开销极小。
    

2. 进程控制块 (PCB) 的作用是什么？

- 作为进程存在的**唯一标志**。
    
- 记录操作系统所需的、用于描述进程的当前情况以及控制进程运行的全部信息。
    
- 使一个在多道程序环境下不能独立运行的程序，成为一个能独立运行的基本单位（或与其他进程并发执行的进程）。
    

3. 为什么需要重定位？为什么要引入动态重定位？

- **需要重定位的原因**：用户程序编译后生成的是逻辑地址，而程序实际装入内存时，必须存放在真实的物理地址中。逻辑地址与物理地址通常不一致，必须进行转换。
    
- **引入动态重定位的原因**：静态重定位在装入时一次性完成地址转换，之后程序在内存中不能移动。引入动态重定位（在程序真正执行时才转换）可以允许程序在内存中移动，极大地便利了内存的紧凑（碎片整理）以及程序和数据的共享。
    

4. 简述页面置换算法中的简单 Clock 置换算法。

- 为每个页面设置一个**访问位**，并将内存中的页面链接成一个**循环队列**。
    
- 当某页被访问时，其访问位置为 1。
    
- 置换时，指针开始循环扫描。如果遇到访问位为 0 的页面，直接将其置换出去；如果遇到访问位为 1 的页面，则将其置为 0，并继续扫描下一个页面，直到找到一个访问位为 0 的页面为止。
    

5. 简述 I/O 中断处理程序的主要处理过程。

- 唤醒被阻塞的 I/O 驱动程序。
    
- 保护被中断进程的 CPU 环境。
    
- 分析中断原因，转入相应的设备中断处理程序进行处理。
    
- 处理完毕后，恢复被中断进程的 CPU 环境，继续执行。
    

6. 举例说明优先级倒置问题，并说明如何解决。

- **什么是倒置**：高优先级进程被低优先级进程延迟或阻塞的现象。
    
- **举例**：进程优先级 A > B > C。C 先运行并锁住了临界资源。A 随后运行，也需要该资源，于是 A 被 C 阻塞。此时中等优先级的 B 抢占了 C 的 CPU 开始运行。这导致 C 无法释放资源，A 只能一直干等。表面上看，中等优先级的 B 间接阻塞了高优先级的 A。
    
- **解决方法**：采用**优先级继承协议**。当高优先级 A 请求 C 占用的资源时，由 C 临时继承 A 的优先级，使其不被 B 抢占，从而尽快执行完并释放资源给 A。
    

7. 简述进程调度的多级反馈队列调度算法。

- 设置多个就绪队列，并为各个队列赋予不同的优先级。第一个队列优先级最高，依次递减。
    
- 赋予各队列中进程执行的时间片大小不同。优先级越高的队列，时间片越小。
    
- 新进程进入内存后，先放入第一队列末尾（FCFS）。如果在一个时间片内未完成，则降级放入第二队列末尾，依此类推。
    
- 仅当高优先级队列为空时，才调度低优先级队列中的进程。
    

8. 简述内存的分页存储管理地址转换过程（无快表）。

- CPU 给出逻辑地址，硬件将其拆分为**页号**和**页内偏移量**。
    
- 硬件将页号与页表寄存器中的**页表长度**进行比较，若大于等于则产生越界中断。
    
- 利用页表起始地址和页号，在内存中查询**页表**，找到对应的**物理块号**。
    
- 将物理块号与页内偏移量拼接，形成最终的**物理地址**，再去内存中访问真正的数据。
    

9. 为什么要引入缓冲区？

- 缓和 CPU 与 I/O 设备间速度不匹配的矛盾。
    
- 减少对 CPU 的中断频率，放宽对 CPU 中断响应时间的限制。
    
- 提高 CPU 和 I/O 设备之间的并行性。
    

### 🎯 概念扫盲：选择/填空/判断题高频考点字典

这里帮你把三套卷子考查的细碎概念按章节进行了归类整理：

**【操作系统基础】**

- **OS 四大特征**：并发、共享、虚拟、异步性 。
    
- **分时系统特征**：多路性、独立性、及时性、交互性 。采用**时间片轮转**调度算法 。
    
- **实时系统**：能及时处理由过程控制反馈的数据并做出响应 。
    
- **双重状态**：为保护 OS 关键数据，处理机状态分为**用户态**和**内核态（管态/核心态）** 。
    

**【进程与并发管理】**

- **进程状态转换**：运行过程中因需要输出数据或打开文件，会进入**阻塞状态**；完成后进入**就绪状态** 。**不可能发生**的转换是：就绪 -> 阻塞 。
    
- **PCB 内容**：包含指令计数器、进程状态、页表始址等，但**不包含 FCB** 。
    
- **调度算法辨析**：
    
    - **EDF 算法**：选择**截止时间最早**的进程 。
        
    - **高响应比优先**：既兼顾短作业，又兼顾长作业等待时间 。
        
    - 多道批处理优先调度：选择 **CPU 与 I/O 均衡型**的作业 。
        
    - 作业调度来源：从**后备状态**的队列中选择 。
        
- **抢占调度原则**：时间片原则、短进程优先原则、**优先级（优先权）原则** 。
    
- **同步与互斥**：
    
    - 同步四准则：空闲让进、忙则等待、有限等待、**让权等待** 。
        
    - 信号量 `S.Value = -3` 表示等待队列中有 **3 个等待进程** 。
        
    - 5个进程共享允许2个进入的互斥段，信号量最小值为 **-3** 。
        
- **死锁**：产生原因是资源不足且**进程推进顺序不当** 。解除死锁的方法有**剥夺资源**和**撤销进程** 。
    
- **临界区判断题大坑**：进程被中断在临界区内，**并未释放 CPU 的所有权**，其他就绪进程可以被调度，但不能进入该临界区。不会永远等待下去（有限等待原则） 。
    

**【内存管理】**

- **程序的链接**：静态链接、装入时动态链接、**运行时动态链接** 。适用于动态链接的存储管理方式是**分段** 。
    
- **动态分区分配**：使得空闲区分布较均匀的是**循环首次适应算法** 。其他还包括首次适应、最佳适应和最坏适应 。
    
- **分页系统**：地址变换由硬件（地址转换机构）完成 。页表寄存器中的页表长度用于判断是否越界 。内存保护包括存取控制和**地址越界保护** 。
    
- **虚拟存储**：提高内存利用率主要通过**虚拟存储技术** 。两种调页策略是**请求调页**和**预调页** 。已经运行过的页面主要是从**对换区**调入 。
    

**【文件与 I/O 设备管理】**

- **文件系统**：
    
    - 控制和管理文件属性信息的是 **FCB** 。
        
    - UNIX 逻辑结构采用**流式文件** 。目录项包括文件名及**索引结点 (i-node) 指针** 。
        
    - 访问方式有**顺序访问**和**随机（直接）访问** 。
        
- **I/O 控制方式**：
    
    - 打印机的 I/O 控制采取**程序中断**方式 。
        
    - DMA 允许 **I/O 设备与内存**之间直接进行数据交换 。
        
    - 通道是一种特殊的**处理机** 。
        
    - 磁盘高速缓存设在**内存**中 。
        
- **SPOOLing 判断题大坑**：由输入输出井、输入输出进程以及**输入输出缓冲区**构成（不只是四个部分）。输出设备忙时，程序**继续**将数据输出到输出井（无需停止）。它是对脱机 I/O 的模拟，且输入设备空闲时可以**提前**输入数据 。
