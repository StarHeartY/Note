# 存储系统

## 辅存


### 得分点一：必考的“硬盘数数”和“算时间”

期末考如果在这章出大题，90% 都是让你算**平均存取时间**。公式非常固定，直接套用下面这个：

$$\text{平均存取时间} = \text{寻道时间} + \text{旋转延迟（转半圈的时间）} + \text{传输时间}$$

> **期末考经典例题：**
> 
> 某机械硬盘转速为 $7200\text{ 转/分}$，平均寻道时间为 $8\text{ ms}$，每个磁道有 100 个扇区。求读一个扇区的平均存取时间。

**标准解题三步法（拿稳步骤分）：**

1. **寻道时间**：题目直接给了，就是 $8\text{ ms}$。
    
2. **旋转延迟**：默认算硬盘**转半圈**的时间。
    
    - 先算转一圈要多久：$7200\text{ 转} / 60\text{ 秒} = 120\text{ 转/秒}$。一圈就是 $\frac{1}{120}\text{ 秒} \approx 8.33\text{ ms}$。
        
    - 转半圈（等待时间）= $8.33\text{ ms} \times 0.5 \approx 4.17\text{ ms}$。
        
3. **传输时间**：读一个扇区，就是转 $\frac{1}{100}$ 圈的时间。
    
    - 一圈是 $8.33\text{ ms}$，所以传输时间 = $\frac{8.33\text{ ms}}{100} \approx 0.083\text{ ms}$。
        

**最后相加**：$8 + 4.17 + 0.083 = 12.253\text{ ms}$。搞定！

### 得分点二：选择/填空题必背的“大白话常识”

如果出选择题或填空题，老师最喜欢捞书本上的原话。你只要记住这 4 句结论：

- **磁头和盘片的关系**：机械硬盘通常是“一个盘面对应一个磁头”，所有磁头连在同一个机械臂上，**一起动、一起停**。
    
- **SSD 固态硬盘的致命痛点**：四个字——**擦完才能写**。它不能直接覆盖旧数据，想改数据得先把那一块擦干净，所以写速度比读速度慢得多。
    
- **什么是柱面**：所有盘面上，半径相同的磁道合起来就叫**柱面**。磁头是不需要频繁前后移动的，读完一个面的这一圈，可以直接电子切换读下一个面的这一圈。
    
- **RAID 是干嘛的**：就是把好几个便宜的硬盘拼起来用，**RAID 0 追求速度（无备份），RAID 1 追求安全（全备份）**。
    


## cache

### 考点一：Cache 的地址映射（核心大题，必考！）

这是期末考最爱出的大题。CPU 给出自主存的物理地址，问你这个地址在 Cache 里怎么对应。一共有三种映射方式：

#### 1. 直接映射（Direct Mapping）

- **规矩**：主存里的某一块，**固定只能放在 Cache 的某一个特定位置**。公式是 $Cache 块号 = 主存块号 \pmod{Cache 总块数}$。
    
- **地址划分**：主存地址会被拆成三段：**[ 主存字块标记 (Tag) ] [ Cache字块地址 (Index) ] [ 字块内地址 (Offset) ]**。
    
- **期末常考**：给你一个十六进制主存地址，问你它映射到 Cache 的哪一行（就是看中间那段 Index 是多少）。
    

#### 2. 全相联映射（Fully Associative Mapping）

- **规矩**：最自由，主存里的任何一块，**可以放到 Cache 的任何一个空闲位置**。
    
- **地址划分**：因为位置不固定，地址只拆成两段：**[ 主存字块标记 (Tag) ] [ 字块内地址 (Offset) ]**。
    

#### 3. 组相联映射（Set Associative Mapping）——最常考！

- **规矩**：前两种的折中（路数固定）。先把 Cache 分成若干个“组”（Set），主存的一块**固定只能去某一个组，但进了组之后可以随便挑位置**。
    
- **地址划分**：拆成三段：**[ 主存字块标记 (Tag) ] [ 组地址 (Set Index) ] [ 字块内地址 (Offset) ]**。
    

> 💡 **期末秒杀口诀**：无论什么映射，**“字块内地址”的位数只由“块大小”决定**；而中间那段的位数，直接看 Cache 一共有多少行（直接映射）或者多少组（组相联）。

### 考点二：Cache 性能指标计算（高频选择/填空题）

这类题目只要死记 3 个环环相扣的公式，公式里通常包含：命中率（$h$）、主存访问时间（$t_m$）、Cache 访问时间（$t_c$）。

1. **平均访问时间 ($T_a$)**：
    
    $$T_a = h \times t_c + (1 - h) \times t_m$$
    
    _(意思就是：命中了就走 Cache 时间，没命中就走主存时间)_
    
2. **Cache-主存系统的效率 ($\cap$)**：
    
    $$\cap = \frac{t_c}{T_a} \times 100\%$$
    
    _(用最快的 Cache 时间除以实际的平均时间，效率越高说明命中率越牛)_
    
3. **加速比**：就是 $\frac{t_m}{T_a}$（原来纯主存的时间比上现在混合系统的时间）。
    

### 考点三：Cache 写策略与替换算法（选择题概念）

当 Cache 满了或者 CPU 要往里写数据时，该怎么办？期末考最喜欢考概念配对：

#### 1. 写策略（数据怎么保持一致？）

- **写回法（Write-back）**：CPU 修改了 Cache 的数据后**不立刻写回主存**，只有当这一块要被换出 Cache 时，才一次性写回主存。速度快，但有风险。
    
- **全写法/写直达法（Write-through）**：CPU 修改 Cache 的同时，**立刻也把主存改了**。安全，但因为主存慢，所以整体变慢。
    

#### 2. 替换算法（Cache 满了踢走谁？）

- **LRU（近期最少使用）**：看谁**最久没有被访问过**，就踢走谁。期末考的最爱，经常让你手动模拟一次 LRU 的替换过程。
    
- **FIFO（先进先出）**：谁最先来就踢走谁。容易出“贝拉迪异常”（Cache 变大命中率反而下降）。
    

### 📝 期末复习建议

你可以去翻一下你们学校前几年的期末真题，找到存储器大题。如果是考 Cache，**绝对是给你一串主存地址访问序列（比如 0, 4, 8, 2...），然后让你用直接映射或组相联映射，写出哪些命中了（Hit），哪些没命中（Miss），最后求命中率**。
