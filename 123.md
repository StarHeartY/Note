# 高中校友升学经验库 — 项目解析

> 本文档为对项目的全面梳理，供后续整理 README 参考。

---

## 一、项目是什么

这是一个面向**高中在校生和毕业校友的公益性静态经验资料库**，目标是沉淀高考备考、志愿填报、大学专业体验、大学生活与发展路径等真实经验，将分散的个人经验转化为结构化、可检索、可持续积累的信息资产。

部署域名：[hz.startyi.com](https://hz.startyi.com)，通过 GitHub Pages 托管。

---

## 二、用了什么技术

| 层次 | 技术选型 |
|---|---|
| **框架** | Astro v6.4.5（静态站框架） |
| **类型系统** | TypeScript v6.0.3 |
| **内容管理** | Astro Content Collections + Markdown/MDX |
| **内容校验** | Zod Schema（内置于 Astro） |
| **样式** | 纯 CSS（无 UI 框架，自写响应式布局） |
| **部署** | GitHub Actions → GitHub Pages |
| **版本控制** | Git + GitHub（两人协作） |
| **搜索** | 预留 Pagefind（尚未接入） |
| **投稿** | 外部表单（飞书/腾讯问卷等，通过环境变量配置链接） |

**明确不引入**：数据库、登录系统、评论、后台管理、AI 问答、动态推荐。

## 四、各文件之间的关系

```
                         astro.config.mjs (站点配置)
                                │
                        src/env.d.ts (类型声明)
                                │
              ┌─────────────────┼─────────────────┐
              │                 │                  │
    src/content.config.ts  src/styles/       src/layouts/
    (内容 schema 定义)     global.css       BaseLayout.astro
              │            (全局样式)        (基础布局模板)
              │                 │                  │
    src/content/articles/      │         ┌────────┴────────┐
    (Markdown 文章源文件)       │         │                 │
              │                 │    src/pages/       src/components/
              │                 │    (各页面路由)      ArticleCard.astro
              │                 │         │           (文章卡片组件)
    src/lib/articles.ts ←──────┴─────────┘                 │
    (工具函数：排序、URL等)                                  │
              │                                             │
    src/data/categories.ts                                  │
    (分类静态数据)                                           │
```

**核心数据流**：

1. Markdown 文章放在 `src/content/articles/`，头部使用 YAML frontmatter 定义元数据
2. `src/content.config.ts` 用 Zod 定义文章 schema，校验所有字段
3. 页面通过 `getCollection("articles")` 获取文章数据
4. `src/lib/articles.ts` 提供排序、格式化、URL 生成等工具函数
5. `ArticleCard.astro` 组件被首页和文章列表页复用
6. 所有页面套用 `BaseLayout.astro` 布局（统一的 header + nav + footer）

## 五、页面结构

| 路由 | 文件 | 功能 |
|---|---|---|
| `/` | `src/pages/index.astro` | **首页**：Hero 区 + 分类网格 + 精选文章 + 最新更新 |
| `/articles/` | `src/pages/articles/index.astro` | **文章列表页**：按更新时间倒序展示全部文章 |
| `/articles/[slug]/` | `src/pages/articles/[slug].astro` | **文章详情页**：正文 + 作者信息卡 + 标签 + 免责声明 |
| `/submit/` | `src/pages/submit.astro` | **投稿页**：投稿说明 + 外部表单入口（链接可配置） |
| `/search/` | `src/pages/search.astro` | **搜索页**：占位入口，引导到文章列表（后续接入 Pagefind） |
| `/about/` | `src/pages/about.astro` | **关于页**：项目说明、内容来源、维护方式 |
| `/disclaimer/` | `src/pages/disclaimer.astro` | **免责声明页**：明确经验参考边界 |
| `/404` | `src/pages/404.astro` | **404 页面**：引导返回首页或浏览文章 |

**导航栏**：首页 / 文章 / 搜索 / 投稿 / 关于

| 分类 | 说明 |

## 七、内容模型（Article Schema）

每篇文章通过 frontmatter 定义以下结构化字段：

- **基础信息**：`title`、`description`、`date`、`updated`
- **分类标签**：`category`（6 选 1）、`tags`（多标签数组）
- **作者信息**：`name`、`graduationYear`、`university`、`major`、`anonymous`、`contactVisible`
- **受众定位**：`audience`（适合读者群体）
- **审核追踪**：`review.status` — draft → submitted → editing → reviewing → approved → published → archived
- **展示控制**：`display.featured`（是否精选）、`display.showDisclaimer`（是否展示免责声明）

## 八、内容审核与发布流程

```
投稿者提交外部表单
    ↓
维护者初筛
    ↓
维护者整理格式
    ↓
必要时回访投稿者确认
    ↓
转换为 Markdown 文章
    ↓
创建 Git 分支 → 提交 PR
    ↓
另一名维护者审核
    ↓
合并到 main
    ↓
GitHub Actions 自动部署到 GitHub Pages
```
